    1. In `src/vcompany/supervisor/supervisor.py`:
       - Add `on_health_change: Callable[[HealthReport], Awaitable[None]] | None = None` parameter to `__init__`
       - Store as `self._on_health_change = on_health_change`
       - In `_make_state_change_callback`, after storing the report AND after the `_restarting` check, add notification logic:
         ```python
         # Notify on significant transitions (not during restarts)
         if self._on_health_change is not None and report.state in ("errored", "running", "stopped"):
             try:
                 loop = asyncio.get_running_loop()
                 loop.create_task(self._on_health_change(report))
             except RuntimeError:
                 pass  # No running event loop
         ```
         This goes AFTER the `if self._restarting: return` block so notifications are suppressed during bulk restarts.
       - Add `import asyncio` at the top (already present)

    2. In `src/vcompany/supervisor/company_root.py`:
       - Add `health_tree()` method that returns `CompanyHealthTree`:
         ```python
         def health_tree(self) -> CompanyHealthTree:
             project_trees = []
             for _project_id, ps in self._projects.items():
                 project_trees.append(ps.health_tree())
             return CompanyHealthTree(
                 supervisor_id=self.supervisor_id,
                 state=self._state,
                 projects=project_trees,
             )
         ```
       - Import `CompanyHealthTree` from `vcompany.container.health`
       - Accept `on_health_change` in `__init__` and pass through to `super().__init__()` -- but since CompanyRoot uses a custom __init__ that calls super() with keyword args, add `on_health_change` to the super() call.
