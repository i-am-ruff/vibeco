  <behavior>
    - Supervisor(supervisor_id, strategy, child_specs, max_restarts=3, window_seconds=600, parent=None, on_escalation=None, data_dir=...)
    - start() starts all children in spec order, wraps each in asyncio.Task for monitoring
    - stop() stops all children in reverse spec order, cancels monitor tasks
    - When a child enters ERRORED state, on_state_change callback notifies supervisor
    - Supervisor sets a _restarting flag during restart to suppress cascade callbacks
    - one_for_one: restart only failed child (stop old, create+start new from spec)
    - all_for_one: stop ALL children reverse order, restart ALL forward order
    - rest_for_one: stop failed+later children reverse order, restart forward order
    - RestartPolicy.TEMPORARY children are never restarted
    - RestartPolicy.TRANSIENT children only restarted if in errored state
    - RestartPolicy.PERMANENT children always restarted
    - Before each restart, check RestartTracker.allow_restart()
    - If tracker returns False, call _escalate() which stops all children and calls parent.handle_child_escalation() or on_escalation callback
    - Supervisor has a .state property returning "running" or "stopped" for health tree compat
    - Supervisor has a .children property returning dict of child_id -> AgentContainer
  </behavior>

       - Internal state: `_children: dict[str, AgentContainer]`, `_tasks: dict[str, asyncio.Task]`, `_child_events: dict[str, asyncio.Event]`, `_restart_tracker: RestartTracker`, `_restarting: bool = False`, `_state: str = "stopped"`
       - `_make_state_change_callback(child_id)`: returns a callback that sets `_child_events[child_id]` when container enters "errored". MUST check `self._restarting` flag and skip if True (prevents cascade in all_for_one/rest_for_one).
       - `async start()`: iterate child_specs in order, call `_start_child(spec)` for each, set `_state = "running"`
       - `async _start_child(spec)`: create AgentContainer.from_spec with on_state_change callback, await container.start(), store in _children and create asyncio.Task for _monitor_child
       - `async _monitor_child(child_id)`: loop awaiting _child_events[child_id], on event check container.state == "errored", call _handle_child_failure
       - `async _handle_child_failure(child_id)`: check RestartPolicy (TEMPORARY=skip, TRANSIENT=only if errored, PERMANENT=always), check _restart_tracker.allow_restart() (if False call _escalate and return), dispatch to strategy method
       - `async _restart_one(failed_id)`: stop old container (await stop()), create+start new from spec
       - `async _restart_all(failed_id)`: set _restarting=True, stop ALL children in reverse spec order, start ALL children in forward spec order, set _restarting=False
       - `async _restart_rest(failed_id)`: find failed index in spec order, set _restarting=True, stop failed+later children in reverse order, restart in forward order, set _restarting=False
       - `async _escalate()`: stop all children, if parent call `await parent.handle_child_escalation(self.supervisor_id)`, else if on_escalation call it with escalation message
       - `async handle_child_escalation(child_supervisor_id)`: handle escalation from a child supervisor (treat like child failure in own strategy)
       - `async stop()`: cancel all monitor tasks, stop all children reverse order, set `_state = "stopped"`
       - `state` property: return `_state`
       - `children` property: return `_children` dict

       CRITICAL implementation details:
       - Always `await old_container.stop()` BEFORE creating new container for same child_id (SQLite file locking)
       - Stop in reverse spec order, start in forward spec order (dependency ordering)
       - The _restarting flag prevents on_state_change callbacks during supervisor-initiated stops from triggering recursive restarts (Pitfall 2 from research)
