### The Gap
The supervisor creates AgentContainer instances but never bridges them to real tmux sessions. The container says "running" because the FSM was transitioned, not because a tmux pane exists. The health tree aggregates these phantom states faithfully.

<specifics>
## Specific Ideas

The fix is bridging AgentContainer lifecycle to TmuxManager:
- Container.start() → TmuxManager.create_session() + send GSD command
- Container health_report() → check tmux pane liveness
- Container.stop() → TmuxManager.kill_session()
- Supervisor monitors tmux liveness, transitions container to ERRORED on crash

</specifics>
