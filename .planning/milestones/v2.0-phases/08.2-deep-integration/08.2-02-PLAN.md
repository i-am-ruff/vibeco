7. **Update `/dispatch`** (dispatch_cmd method):
   - When container is found and state is "running": check `container.is_tmux_alive()`. If alive, report "running (tmux alive)". If dead, report "running (tmux dead, will be detected by liveness monitor)".
   - When container is found and state is "stopped" or "errored": trigger restart through the supervisor. Find the ProjectSupervisor that owns this container, then call the supervisor's _restart_one or use `await container.error()` to trigger the monitor/restart cycle. Simplest approach: find the project supervisor, get the child spec, and call `await project_sup._start_child(spec)`:
     ```python
     container = await self.bot.company_root._find_container(agent_id)
     if container is not None:
         state = container.state
         if state == "running":
             tmux_status = "tmux alive" if container.is_tmux_alive() else "tmux dead"
             await interaction.followup.send(
                 f"Agent **{agent_id}** is {state} ({tmux_status})."
             )
         elif state in ("stopped", "errored"):
             # Find ProjectSupervisor and restart
             for ps in self.bot.company_root.projects.values():
                 if agent_id in ps.children:
                     spec = ps._get_spec(agent_id)
                     if spec:
                         await ps._start_child(spec)
                         await interaction.followup.send(
                             f"Agent **{agent_id}** restarted via supervision tree (new tmux session)."
                         )
                     break
             else:
                 await interaction.followup.send(
                     f"Agent **{agent_id}** is {state} but not found in any project supervisor."
                 )
         else:
             await interaction.followup.send(
                 f"Agent **{agent_id}** container state: {state}."
             )
     ```
   - For agent_id == "all": iterate all projects, all children, report states with tmux liveness.
