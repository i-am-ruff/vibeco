**1. Convert CommandsCog to slash commands** (`src/vcompany/bot/cogs/commands.py`):

- Add imports: `from discord import app_commands` and `from vcompany.bot.permissions import is_owner_app_check`
- Replace `cog_check` with `async def interaction_check(self, interaction: discord.Interaction) -> bool` that checks `self.bot.is_bot_ready` and responds via `interaction.response.send_message("Bot is starting up...", ephemeral=True)` if not ready
- For EVERY command, convert from prefix to slash:
  - `@commands.command(name="X")` -> `@app_commands.command(name="X", description="...")`
  - `@is_owner()` -> `@is_owner_app_check()`
  - `ctx: commands.Context` -> `interaction: discord.Interaction`
  - `await ctx.send(...)` -> `await interaction.response.send_message(...)` for first response, `await interaction.followup.send(...)` for subsequent
  - `ctx.guild` -> `interaction.guild`
  - `ctx.author` -> `interaction.user`
  - `ctx.channel` -> `interaction.channel`
  - `ctx.message` -> not available (slash commands have no message object)

- For each project-dependent command (dispatch, status, kill, relaunch, standup, integrate, new-project), add early guard:
  ```python
  if self.bot.project_config is None:
      await interaction.response.send_message(
          "No project loaded. Ask the Strategist in #strategist to help you set one up.",
          ephemeral=True
      )
      return
  ```

- Specific conversion notes per command:
  - **new-project**: `name` becomes `@app_commands.describe(name="Project name")` parameter. Use `interaction.response.defer()` before channel setup (may be slow), then `interaction.followup.send()`. Remove thread creation from ctx.message (no message object for slash commands).
  - **dispatch**: `agent_id` becomes a required string parameter with `@app_commands.describe`. Use `interaction.response.defer()` before dispatch_all (slow).
  - **status**: Use `interaction.response.defer()` before generate_project_status, then `interaction.followup.send(embed=embed)`.
  - **kill**: Use `interaction.response.send_message()` for the ConfirmView. Note: `view.interaction_user_id = interaction.user.id`.
  - **relaunch**: `agent_id` required parameter. `interaction.response.defer()` before relaunch.
  - **standup**: `interaction.response.defer()` before thread creation loop.
  - **integrate**: `interaction.response.send_message()` for ConfirmView, then `interaction.followup.send()` for results.

- The `on_message` listener for standup thread routing stays as-is (it uses on_message, not a command).
- The `_on_checkin` and `wire_monitor_callbacks` methods stay as-is (internal callbacks).

- The StrategistCog has no prefix commands (it uses on_message listener), so no command conversion needed.
