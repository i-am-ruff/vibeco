**conversation.py** — Replace AsyncAnthropic-based conversation with Claude CLI persistent session:

1. Remove the `AsyncAnthropic` client parameter from `__init__`. Instead accept optional `model: str = "opus"` (claude CLI model alias) and `persona_path`.
2. On first `send()`, start a claude CLI session using `asyncio.create_subprocess_exec`:
   - Command: `claude -p --model {model} --system-prompt "{system_prompt}" --output-format stream-json --session-id {uuid}`
   - Generate a stable UUID4 session ID in `__init__` and store as `self._session_id`.
   - For subsequent sends, use `claude -p --resume {session_id} --output-format stream-json` to continue the conversation. The `--resume` flag reconnects to the same session, preserving conversation history (this replaces the manual messages array).
3. Parse `stream-json` output line-by-line. Each line is JSON; look for `{"type":"assistant","content":[{"type":"text","text":"..."}]}` in the final result or `{"type":"content_block_delta","delta":{"type":"text_delta","text":"chunk"}}` during streaming. Yield text chunks as they arrive.
4. Remove `_maybe_check_tokens` and `_perform_knowledge_transfer` methods entirely. Claude CLI manages its own context window. Remove the `_messages` list, `_total_tokens`, `_message_count_since_check` fields. Keep the `asyncio.Lock` for send serialization.
5. Keep `_load_persona()` static method unchanged.
6. Update `messages` property to return empty list (or remove it). Update `token_count` property to return 0 (or remove it). These were only used for KT which is now handled by CLI internally.
7. The `send()` method signature stays the same: `async def send(self, content: str, role: str = "user") -> AsyncGenerator[str, None]`.

**pm.py** — Replace AsyncAnthropic with `claude -p` subprocess:

1. Remove `AsyncAnthropic` import and `client` parameter from `__init__`. Accept only `project_dir: Path` and optionally `model: str = "sonnet"`.
2. Replace `_answer_directly()` to use `asyncio.create_subprocess_exec`:
   - Command: `claude -p --model {model} --system-prompt "{system_prompt}" --output-format json --allowedTools ""`
   - Pass the question via stdin.
   - Parse JSON output: `{"type":"result","result":"answer text"}`. Extract the result text.
   - Use `--allowedTools ""` to disable all tools (PM should only answer, not run code).
3. Keep all confidence scoring, escalation logic, context loading, and decision log loading exactly the same.
4. Error handling: catch `subprocess` errors and return the same fallback message "PM was unable to generate an answer. Escalating."

**config.py** — Make `anthropic_api_key` no longer relevant for PM/Strategist:

1. Keep the `anthropic_api_key` field in BotConfig (backward compat) but it will no longer gate PM/Strategist initialization.
