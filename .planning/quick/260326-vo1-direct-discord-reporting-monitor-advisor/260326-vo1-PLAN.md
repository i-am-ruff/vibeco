**report_cmd.py — complete rewrite:**

Replace file-based reporting with direct Discord HTTP API posting. The agent calls `vco report "starting phase 1"` and it posts directly to #agent-{AGENT_ID} in Discord.

Implementation:
1. Read env vars: `DISCORD_BOT_TOKEN`, `DISCORD_GUILD_ID`, `AGENT_ID` (all set at dispatch time).
2. Use `httpx` (sync, already a project dependency) to:
   a. GET `https://discord.com/api/v10/guilds/{guild_id}/channels` with `Authorization: Bot {token}` header to find the channel named `agent-{agent_id}`.
   b. POST `https://discord.com/api/v10/channels/{channel_id}/messages` with `{"content": "{timestamp} {agent_id}: {status_text}"}`.
3. Cache the channel_id after first lookup (module-level dict keyed by agent_id) so subsequent calls skip the channel list API call.
4. On any HTTP error, fall back to printing to stderr (agent must not hang on Discord failures). Use a 5-second timeout on httpx calls.
5. Keep the click command signature identical: `@click.command()` with `@click.argument("status", nargs=-1, required=True)`.
6. Remove all imports/references to `PROJECTS_BASE`, `Path`, file writing. No file I/O at all.

**bot/cogs/commands.py — remove file relay, add /toggle-advisories and advisory posting:**

1. Delete `_on_agent_report()` method entirely.
2. In `wire_monitor_callbacks()`, remove the line `monitor._on_agent_report = self._on_agent_report`.
3. Add a `self._advisories_enabled: bool = True` attribute in `__init__`.
4. Add a new async method `_on_advisory(self, agent_id: str, message: str) -> None`:
   ```python
   async def _on_advisory(self, agent_id: str, message: str) -> None:
       if not self._advisories_enabled:
           return
       guild = self.bot.get_guild(self.bot._guild_id)
       if not guild:
           return
       channel = discord.utils.get(guild.text_channels, name="strategist")
       if channel:
           try:
               await channel.send(f"[monitor-advisory] {message}")
           except Exception:
               logger.exception("Failed to post advisory")
   ```
5. In `wire_monitor_callbacks()`, wire the advisory callback:
   ```python
   monitor._on_advisory = self._on_advisory
   ```
6. Add `/toggle-advisories` slash command:
   ```python
   @app_commands.command(name="toggle-advisories", description="Enable/disable monitor advisories to Strategist")
   @is_owner_app_check()
   async def toggle_advisories(self, interaction: discord.Interaction) -> None:
       self._advisories_enabled = not self._advisories_enabled
       state = "enabled" if self._advisories_enabled else "disabled"
       await interaction.response.send_message(f"Monitor advisories are now **{state}**.")
   ```
