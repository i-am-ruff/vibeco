**DockerTransport class:**

Constructor:
```python
def __init__(self, docker_image: str = "vco-agent:latest", project_name: str = "") -> None:
    self._client: docker.DockerClient = docker.from_env()
    self._image = docker_image
    self._project = project_name
    self._sessions: dict[str, _DockerSession] = {}
```
Note: constructor takes `docker_image` and `project_name` from transport_deps. Uses `docker.from_env()` to connect to Docker daemon.

**setup() method** -- per D-04, D-06, D-07, D-08, D-09:
1. Build container name: `f"vco-{self._project}-{agent_id}"` (D-04 deterministic naming)
2. Create `_DockerSession` and store in `self._sessions[agent_id]`
3. Extract `interactive` from kwargs (default True), same as LocalTransport
4. Try `self._client.containers.get(container_name)` to find existing container:
   - If found and status == "running": reuse it, log "Reusing running container" (D-06)
   - If found and status == "exited": call `container.start()`, log "Restarting stopped container" (D-06)
   - Catch `docker.errors.NotFound`: create new container (see below)
5. Container creation (when not found):
   - Derive signal_socket_path: `str(VCO_SOCKET_PATH.parent / "vco-signal.sock")`
   - Build volumes dict:
     ```python
     volumes = {
         str(working_dir): {"bind": "/workspace", "mode": "rw"},          # DOCK-04
         str(VCO_SOCKET_PATH): {"bind": str(VCO_SOCKET_PATH), "mode": "rw"},  # DOCK-03
         signal_sock: {"bind": signal_sock, "mode": "rw"},                     # DOCK-03
     }
     ```
   - Build environment dict:
     ```python
     environment = {
         "ANTHROPIC_API_KEY": os.environ.get("ANTHROPIC_API_KEY", ""),
         "VCO_AGENT_ID": agent_id,
     }
     ```
   - Call `self._client.containers.create(image=self._image, name=container_name, volumes=volumes, environment=environment, stdin_open=True, tty=True, network_mode="none", working_dir="/workspace")`
   - Call `container.start()`
6. After start (all paths): call `container.reload()` to refresh status (pitfall 5)
7. Store `container.id` in session
8. If interactive: exec into container to start tmux session:
   ```python
   await asyncio.to_thread(
       container.exec_run, ["tmux", "new-session", "-d", "-s", "main"]
   )
   ```
   This addresses pitfall 6 -- tmux server must be running before send-keys.
9. All docker-py calls wrapped in `asyncio.to_thread()` (pitfall 1, same pattern as LocalTransport with libtmux).

**teardown() method** -- per D-05:
1. Pop session from `self._sessions`
2. Get container by `session.container_id`
3. Call `container.stop()` (D-05: stop but keep container)
4. Log "Stopped container {container_name}, kept for restart"
5. Wrap in try/except for `docker.errors.NotFound` (container already removed)

**exec() method** -- per D-10:
1. Get session, raise RuntimeError if not found
2. Get container via `self._client.containers.get(session.container_id)`
3. If `session.interactive`:
   - Build command string from `command` (join list with spaces if list)
   - `container.exec_run(["tmux", "send-keys", "-t", "main", cmd_str, "Enter"], detach=True)`
   - Return ""
4. If piped (not interactive):
   - Build command list from `command` (split if string)
   - `exit_code, output = container.exec_run(cmd_list, workdir=session.container_workdir)`
   - If exit_code != 0: raise RuntimeError with truncated output
   - Return `output.decode() if output else ""`
5. All docker-py calls wrapped in `asyncio.to_thread()`

**exec_streaming() method** -- async generator for piped mode:
1. Get session, raise RuntimeError if not found
2. If interactive: return (yield nothing, same as LocalTransport)
3. Get container
4. Use `container.exec_run(command, stream=True, workdir=session.container_workdir)` -- returns `(exit_code, generator)`
5. Bridge sync generator to async using a helper: iterate the sync generator in a thread executor, yielding decoded chunks:
   ```python
   loop = asyncio.get_event_loop()
   _, output_gen = await asyncio.to_thread(
       container.exec_run, command, stream=True, workdir=session.container_workdir
   )
   # Iterate sync generator in executor
   while True:
       chunk = await loop.run_in_executor(None, next, output_gen, None)
       if chunk is None:
           break
       yield chunk.decode() if isinstance(chunk, bytes) else chunk
   ```
   This addresses pitfall 2 (sync generator bridging).

**is_alive() method** -- per D-11, synchronous:
1. Get session, return False if not found
2. Try to get container: `self._client.containers.get(session.container_id)`
3. Call `container.reload()` (pitfall 5: stale status)
4. Layer 1: if `container.status != "running"`, return False
5. Layer 2: `exit_code, _ = container.exec_run(["tmux", "has-session", "-t", "main"])`
6. Return `exit_code == 0`
7. Catch `docker.errors.NotFound`: return False

**send_keys() method:**
1. Get session, return if not found or not interactive
2. Get container
3. Build keys command for tmux. If `enter` is True, append "Enter" to tmux send-keys:
   ```python
   cmd = ["tmux", "send-keys", "-t", "main", keys]
   if enter:
       cmd.append("Enter")
   await asyncio.to_thread(container.exec_run, cmd, detach=True)
   ```

**read_file() method** -- volume-based (per research Pattern 6):
1. Get session
2. Resolve host path: if `path` is relative or starts with container_workdir, map to host working_dir:
   ```python
   def _resolve_host_path(self, session: _DockerSession, path: Path) -> Path:
       path_str = str(path)
       if path_str.startswith(session.container_workdir):
           relative = path_str[len(session.container_workdir):].lstrip("/")
           return session.working_dir / relative
       if not path.is_absolute():
           return session.working_dir / path
       return path  # Assume host path if absolute and not in container
   ```
3. `return await asyncio.to_thread(host_path.read_text)`

**write_file() method** -- volume-based:
1. Same path resolution as read_file
2. `await asyncio.to_thread(host_path.write_text, content)`
