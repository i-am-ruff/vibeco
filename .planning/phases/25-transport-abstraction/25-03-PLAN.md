```python
def create_container(
    spec: ChildSpec,
    data_dir: Path,
    comm_port: object | None = None,
    on_state_change: Callable[[HealthReport], None] | None = None,
    transport: AgentTransport | None = None,
    transport_deps: dict | None = None,
    project_dir: Path | None = None,
    project_session_name: str | None = None,
) -> AgentContainer:
    """Create the correct AgentContainer subclass for a ChildSpec.

    Transport resolution (per D-07):
    1. If `transport` is provided, use it directly (override/testing).
    2. Otherwise, read `AgentConfig.transport` from spec (defaults to "local"),
       look up the class in _TRANSPORT_REGISTRY, and instantiate with transport_deps.
       New transports = add one line to _TRANSPORT_REGISTRY.

    Args:
        spec: Child specification with agent_type and context.
        data_dir: Root directory for container persistent data.
        comm_port: Optional communication port.
        on_state_change: Optional callback for state transitions.
        transport: Optional pre-built transport (overrides registry lookup).
        transport_deps: Dict of construction kwargs for transport instantiation.
                        For LocalTransport: {"tmux_manager": tmux_manager_instance}.
        project_dir: Optional project directory for clone/prompt paths.
        project_session_name: Optional tmux session name for the project.
    """
    if transport is None:
        # D-07: Look up AgentConfig.transport field, instantiate from registry
        transport_name = getattr(spec, 'transport', 'local')
        # spec might be ChildSpec -- check if it carries the agent config's transport field
        # ChildSpec.context has agent_type but not transport; check spec attributes
        if hasattr(spec, 'config') and hasattr(spec.config, 'transport'):
            transport_name = spec.config.transport
        transport_cls = _TRANSPORT_REGISTRY.get(transport_name)
        if transport_cls is not None:
            deps = transport_deps or {}
            transport = transport_cls(**deps)

    cls = _REGISTRY.get(spec.agent_type, AgentContainer)
    return cls.from_spec(
        spec,
        data_dir=data_dir,
        comm_port=comm_port,
        on_state_change=on_state_change,
        transport=transport,
        project_dir=project_dir,
        project_session_name=project_session_name,
    )
```

This means adding a new transport is truly "add one line to registry" -- the factory handles instantiation.

- If ChildSpec has a `config: AgentConfig` attribute, use `spec.config.transport`
- If ChildSpec only has `context: ContainerContext`, add a `transport` field to ContainerContext (string, default "local")
- Simplest: add `transport: str = "local"` to ChildSpec itself

The executor should read ChildSpec and choose the cleanest path. The key requirement: the factory MUST be able to determine which transport to instantiate from the spec without a pre-built transport being passed in.
