    Create `src/vcompany/resilience/message_queue.py`:

    1. `MessagePriority(IntEnum)` with levels: ESCALATION=0, SUPERVISOR=1, STATUS=2, HEALTH_DEBOUNCED=3. Lower number = higher priority (asyncio.PriorityQueue pops lowest first).

    2. `QueuedMessage` as a `@dataclass(order=True)` with:
       - `priority: int` (compared)
       - `timestamp: float` (compared, for FIFO within same priority)
       - `channel_id: int` (compare=False)
       - `content: str | None` (compare=False, default=None)
       - `embed: object | None` (compare=False, default=None)

    3. `MessageQueue` class:
       - `__init__(self, send_func, max_rate=5.0, debounce_seconds=5.0)` -- send_func is `Callable[[QueuedMessage], Awaitable[None]]` (injected, not a bot reference -- makes testing easy). max_rate is messages/second. debounce_seconds is health coalesce window.
       - `_queue: asyncio.PriorityQueue[QueuedMessage]`
       - `_backoff: float = 0.0`
       - `_health_buffer: dict[int, QueuedMessage]` for debounce
       - `_health_flush_handle: asyncio.TimerHandle | None`
       - `_drain_task: asyncio.Task | None`

       Methods:
       - `async start()` -- creates `_drain_task = asyncio.create_task(self._drain_loop())`
       - `async stop()` -- cancels drain task, flushes health buffer
       - `async enqueue(msg: QueuedMessage)` -- put on priority queue
       - `async enqueue_health(channel_id: int, embed: object)` -- debounced: stores in `_health_buffer[channel_id]`, resets timer, on timer expiry calls `_flush_health()` which moves buffered items to the priority queue
       - `async _drain_loop()` -- infinite loop: get from queue, sleep backoff if nonzero, call send_func, on success reset backoff, on discord.HTTPException with status 429 re-enqueue and double backoff (min(backoff*2, 60.0), starting from 1.0), sleep interval (1/max_rate)
       - `async _flush_health()` -- move all `_health_buffer` entries into `_queue`, clear buffer

    Key design: send_func is a callable injection, NOT a bot reference. This makes the class testable without Discord mocks. In production, the caller wraps `channel.send()` into a send_func. Tests pass a simple mock.

    For the 429 detection: import `discord` and catch `discord.HTTPException` where `e.status == 429`. If discord is not importable in tests, make the exception type configurable or catch a base class.

    Actually, to keep the module decoupled from discord.py, define an approach: the `_drain_loop` calls `send_func(msg)` and catches `Exception`. The send_func itself (provided by the caller) is responsible for translating discord.HTTPException into a `RateLimitError` that MessageQueue understands. OR simpler: MessageQueue catches all exceptions, and send_func raises `MessageQueue.RateLimited` (a custom exception class defined in message_queue.py) when it encounters a 429. This keeps MessageQueue completely Discord-agnostic.

    Define `class RateLimited(Exception): pass` in message_queue.py. The drain loop catches RateLimited to trigger backoff, and catches Exception for general errors (log and discard).

    Create tests in `tests/test_message_queue.py` using pytest-asyncio. Use a mock send_func (async callable that records calls). Test all behaviors listed above.
