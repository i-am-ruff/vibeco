    `DegradedModeManager` class:
    - `__init__(self, health_check: Callable[[], Awaitable[bool]], check_interval: float = 60.0, failure_threshold: int = 3, recovery_threshold: int = 2, on_degraded: Callable[[], Awaitable[None]] | None = None, on_recovered: Callable[[], Awaitable[None]] | None = None)`
    - `health_check` is an injected async callable that returns True if healthy, False if unhealthy. This decouples the manager from the anthropic SDK -- the caller provides the check function.
    - `_state: str = "normal"` (literal "normal" or "degraded")
    - `_consecutive_failures: int = 0`
    - `_consecutive_successes: int = 0`
    - `_check_interval: float`
    - `_failure_threshold: int`
    - `_recovery_threshold: int`
    - `_on_degraded: Callable | None`
    - `_on_recovered: Callable | None`
    - `_task: asyncio.Task | None`

    Methods:
    - `@property is_degraded -> bool` -- returns `_state == "degraded"`
    - `@property state -> str` -- returns _state
    - `async start()` -- creates background task running `_check_loop()`
    - `async stop()` -- cancels the background task
    - `async _check_loop()` -- infinite loop: call health_check(), call `_record_result(healthy)`, sleep check_interval. Wrap health_check in try/except (any exception = unhealthy).
    - `async _record_result(healthy: bool)`:
      If healthy:
        _consecutive_successes += 1
        _consecutive_failures = 0
        If _state == "degraded" and _consecutive_successes >= _recovery_threshold:
          _state = "normal"
          _consecutive_successes = 0
          if _on_recovered: await _on_recovered()
      If not healthy:
        _consecutive_failures += 1
        _consecutive_successes = 0
        If _state == "normal" and _consecutive_failures >= _failure_threshold:
          _state = "degraded"
          _consecutive_failures = 0
          if _on_degraded: await _on_degraded()

    - `async record_operational_failure()` -- passive detection: called by external code when a Claude API call fails during normal operation. Increments _consecutive_failures, resets _consecutive_successes. This enables passive health detection (research recommendation) in addition to active probing.
    - `async record_operational_success()` -- called when a Claude API call succeeds. Increments _consecutive_successes, resets _consecutive_failures.

    Add to `src/vcompany/resilience/__init__.py` (append): export DegradedModeManager.

    Create tests in `tests/test_degraded_mode.py`. Use a mock health_check (async callable returning True/False based on test scenario). Use short check_interval (0.01s) for loop tests. Test all behaviors above.
