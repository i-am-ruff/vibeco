Create two new files:

**channel_transport.py** -- ChannelTransport Protocol:
- `@runtime_checkable` Protocol class with two async methods and one property:
  - `async def spawn(self, agent_id: str, *, config: dict, env: dict[str, str] | None = None, working_dir: str | None = None) -> asyncio.subprocess.Process` -- Spawn vco-worker, return process with stdin/stdout PIPE'd. Caller sends StartMessage and reads WorkerMessages.
  - `async def terminate(self, agent_id: str) -> None` -- Force-terminate the execution environment for an agent.
  - `@property def transport_type(self) -> str` -- Return "native" or "docker".

**native.py** -- NativeTransport:
- Class implementing ChannelTransport.
- `__init__(self)` -- no constructor args needed.
- `spawn()` implementation:
  - Build command: `[sys.executable, "-m", "vco_worker"]`
  - If `working_dir` provided, set `cwd=working_dir` on subprocess.
  - Build env dict: merge `os.environ` with passed `env` dict (if any). Always include `VCO_AGENT_ID=agent_id`.
  - Call `asyncio.create_subprocess_exec(*cmd, stdin=PIPE, stdout=PIPE, stderr=PIPE, cwd=working_dir, env=merged_env)`
  - Track process in `self._processes[agent_id] = process`
  - Return the process.
- `terminate()` implementation:
  - Pop process from `self._processes`. Call `process.terminate()`. Try `wait_for(process.wait(), 5.0)`, then `process.kill()` on timeout.
- `transport_type` property returns `"native"`.
- Add logging via `logging.getLogger("vcompany.transport.native")`.

Create `docker_channel.py` -- DockerChannelTransport:

- Class implementing ChannelTransport.
- `__init__(self, docker_image: str = "vco-agent:latest")` -- store image name.
- `spawn()` implementation:
  - Container name: `f"vco-{agent_id}"`
  - Build command list:
    ```python
    cmd = ["docker", "run", "-i", "--rm", "--name", container_name]
    ```
  - Add `--network none` for isolation.
  - If `env` dict provided, add `-e KEY=VALUE` for each entry. Always add `-e VCO_AGENT_ID={agent_id}`.
  - If `working_dir` provided, add `-v {working_dir}:/workspace -w /workspace`.
  - Append image and entrypoint: `[self._image, "python", "-m", "vco_worker"]`
  - CRITICAL: Do NOT use `-t` flag (no TTY -- would corrupt NDJSON output).
  - CRITICAL: Do NOT mount Unix sockets (no daemon socket, no signal socket).
  - Call `asyncio.create_subprocess_exec(*cmd, stdin=PIPE, stdout=PIPE, stderr=PIPE)`
  - Track in `self._containers[agent_id] = container_name` and `self._processes[agent_id] = process`
  - Return process.
- `terminate()` implementation:
  - Pop from tracking dicts.
  - Run `subprocess.run(["docker", "kill", container_name], capture_output=True)` to kill container.
  - Then `process.terminate()` to clean up the docker client process.
- `transport_type` property returns `"docker"`.
- Add logging via `logging.getLogger("vcompany.transport.docker_channel")`.

Key differences from old DockerTransport:
- NO docker-py SDK for spawning (uses subprocess `docker run -i`).
- NO socket mounts. NO tmux inside container.
- NO `-t` flag (no TTY).
- Uses `--rm` for auto-cleanup.
- Returns process with stdin/stdout -- identical interface to NativeTransport.
  </action>
