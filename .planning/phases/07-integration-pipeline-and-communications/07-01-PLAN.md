    ```python
    async def attribute_failures(
        integration_dir: Path,
        failed_tests: list[str],
        agent_ids: list[str],
    ) -> dict[str, list[str]]:
        """Map test failures to responsible agent branches.

        Algorithm per D-06:
        1. For each agent: checkout main, merge only that agent's branch
        2. Re-run ONLY the failing tests (not full suite)
        3. If tests fail with just agent-A -> agent-A owns those tests
        4. If tests pass with every individual branch -> interaction failure ("_interaction")

        Returns: {agent_id: [test_names], "_interaction": [test_names], "_flaky": [test_names]}
        """
    ```

    Implementation details:
    - For each agent_id, create a temp branch from main: `git checkout -b _attr_{agent_id} main`
    - Merge only that agent's branch: `git merge agent/{agent_id.lower()}`
    - Run only the failing tests: `pytest {test1} {test2} ... --tb=line -q`
    - Track which tests fail for each agent
    - Tests that fail on no individual branch = "_interaction" (cross-agent issue)
    - Tests that pass on full merge re-run = "_flaky"
    - Clean up temp branches after attribution
    - All git/subprocess calls wrapped in asyncio.to_thread
