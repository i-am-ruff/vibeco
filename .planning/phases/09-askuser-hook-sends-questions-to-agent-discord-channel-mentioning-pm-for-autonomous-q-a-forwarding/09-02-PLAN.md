    **New functions (stdlib only per HOOK-06):**

    1. `_make_request(url, bot_token, method="GET", data=None, timeout=10) -> dict | None`:
       Common HTTP helper. Uses urllib.request. Returns parsed JSON or None on error.
       Sets Authorization header to `Bot {bot_token}` and Content-Type to application/json for POST.

    2. `resolve_channel(bot_token, guild_id, agent_id, project_name=None) -> str | None`:
       GET /guilds/{guild_id}/channels. Find channel named `agent-{agent_id}`.
       When project_name provided, only match channels under `vco-{project_name}` category
       (type==4 for categories, check parent_id). Pattern from report_cmd.py.
       Returns channel_id string or None.

    3. `post_question(bot_token, channel_id, agent_id, request_id, questions) -> str | None`:
       POST /channels/{channel_id}/messages with:
       - content: `[{agent_id}] has a question:` (D-05 entity prefix)
       - embed with title `Question from {agent_id}`, description from questions[0]["question"],
         fields from options (name=label, value=description), footer text `Request: {request_id}`,
         color 0x3498DB
       Returns message_id string or None.

    4. `poll_for_reply(bot_token, channel_id, question_msg_id, poll_interval=POLL_INTERVAL, max_polls=MAX_POLLS_PM) -> str | None`:
       GET /channels/{channel_id}/messages?after={question_msg_id}&limit=10
       For each message, check if message_reference.message_id == question_msg_id.
       If found, return message content (the answer).
       If a non-reply message contains ESCALATION_MARKER, switch max_polls to MAX_POLLS_ESCALATION (infinite loop per D-18).
       On HTTP 429 (rate limit), read Retry-After header and sleep that duration.
       Returns answer string or None on timeout.

    5. `alert_timeout(bot_token, channel_id, agent_id, request_id, fallback) -> None`:
       POST a timeout warning message to the channel (not embed, just text with prefix).
       Content: `[system] Timeout: Auto-fallback for {agent_id} (Request: {request_id}): {fallback}`
       Wrapped in try/except, never blocks.

    ```python
    def main():
        input_data = parse_stdin()

        tool_input = input_data.get("tool_input", {})
        questions = tool_input.get("questions", [])
        if not questions:
            output_allow()

        agent_id = os.environ.get("VCO_AGENT_ID", os.environ.get("AGENT_ID", "unknown-agent"))
        bot_token = os.environ.get("DISCORD_BOT_TOKEN", "")
        guild_id = os.environ.get("DISCORD_GUILD_ID", "")
        project_name = os.environ.get("PROJECT_NAME", "")
        timeout_mode = os.environ.get("VCO_TIMEOUT_MODE", "continue")
        request_id = str(uuid.uuid4())

        if not bot_token or not guild_id:
            output_deny("Hook error: DISCORD_BOT_TOKEN and DISCORD_GUILD_ID not set. Auto-fallback.")
            return  # output_deny calls sys.exit

        channel_id = resolve_channel(bot_token, guild_id, agent_id, project_name or None)
        if not channel_id:
            output_deny(f"Hook error: Could not find #agent-{agent_id} channel. Auto-fallback.")
            return

        msg_id = post_question(bot_token, channel_id, agent_id, request_id, questions)

            return

        answer = poll_for_reply(bot_token, channel_id, msg_id)

        if answer is None:
            fallback = get_fallback_answer(questions, timeout_mode)
            if timeout_mode == "continue":
                alert_timeout(bot_token, channel_id, agent_id, request_id, fallback)
            output_deny(fallback)
        else:
            output_deny(f"Answered via Discord: {answer}")
    ```
