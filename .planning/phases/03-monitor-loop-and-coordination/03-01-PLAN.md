    1. Create `src/vcompany/monitor/__init__.py` (empty).

    2. Create `src/vcompany/models/monitor_state.py` with Pydantic models (per D-01 through D-09):
       - `AgentMonitorState`: tracks per-agent state between cycles
         - `agent_id: str`
         - `last_commit_time: datetime | None = None`
         - `last_plan_mtimes: dict[str, float] = {}` (maps plan file path to mtime)
         - `current_phase: str = "unknown"`
         - `phase_status: str = "unknown"`
       - `CheckResult`: result of a single check
         - `check_type: Literal["liveness", "stuck", "plan_gate"]`
         - `agent_id: str`
         - `passed: bool`
         - `detail: str = ""`
         - `new_plans: list[str] = []` (for plan_gate results)

    3. Create `src/vcompany/monitor/checks.py` with three functions per D-02, D-03, D-05/D-09:

       `check_liveness(agent_id: str, tmux: TmuxManager, pane: libtmux.Pane) -> CheckResult`:
       - Call tmux.is_alive(pane). Return CheckResult with alive status.
       - Wrap in try/except, return CheckResult(passed=False, detail=error) on failure.

       `check_stuck(agent_id: str, clone_dir: Path, *, threshold_minutes: int = 30, now: datetime | None = None) -> CheckResult`:
       - Use git_ops.log(clone_dir, args=["--format=%aI", "-1"]) to get latest commit time.
       - Parse ISO timestamp, compare to now. If diff > threshold_minutes, return stuck=True.
       - If no commits or parse failure, treat as stuck.
       - Use `now` parameter injection for testing (same pattern as CrashTracker).
       - Wrap in try/except per MON-01.

       `check_plan_gate(agent_id: str, clone_dir: Path, last_plan_mtimes: dict[str, float]) -> tuple[CheckResult, dict[str, float]]`:
       - Scan clone_dir / ".planning" / "phases" for *-PLAN.md files.
       - Compare each file's stat().st_mtime against last_plan_mtimes dict.
       - If last_plan_mtimes is empty (first run per Research open question 2), seed all current mtimes WITHOUT reporting as new.
       - Return updated mtimes dict along with CheckResult listing any new plans.
       - Wrap in try/except per MON-01.

    4. Create `tests/test_monitor_checks.py` with all behavior tests listed above.
       - Use tmp_path for filesystem fixtures (plan files, clone dirs).
       - Mock TmuxManager for liveness tests.
       - Mock git_ops.log for stuck detection tests.
       - Use `now` parameter injection for time-dependent tests (same as Phase 2 pattern, NOT freezegun).
