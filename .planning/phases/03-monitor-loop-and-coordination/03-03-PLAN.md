    Create `src/vcompany/monitor/loop.py` implementing D-01 through D-09:

    ```python
    class MonitorLoop:
        """Main monitor loop. Checks all agents every 60s cycle."""

        CYCLE_INTERVAL = 60  # seconds, per D-01

        def __init__(
            self,
            project_dir: Path,
            config: ProjectConfig,
            tmux: TmuxManager,
            *,
            on_agent_dead: Callable[[str], None] | None = None,
            on_agent_stuck: Callable[[str], None] | None = None,
            on_plan_detected: Callable[[str, Path], None] | None = None,
            cycle_interval: int | None = None,  # For testing override
        ) -> None:
    ```

    Key implementation details:
    - Store `_agent_states: dict[str, AgentMonitorState]` for per-agent tracking between cycles.
    - Store `_running: bool` for graceful shutdown.
    - Callbacks follow the CrashTracker.on_circuit_open pattern from Phase 2.

    `async def run(self) -> None`: Main loop. Sets `_running = True`, loops `_run_cycle` + `asyncio.sleep(cycle_interval)`.

    `async def _run_cycle(self) -> None`:
    1. Write heartbeat FIRST (per Pitfall 6).
    2. Load agents.json registry to get pane info.
    3. Create per-agent async tasks: `asyncio.gather(*[self._check_agent(a.id) for a in config.agents], return_exceptions=True)`.
    4. Log any returned exceptions but do NOT propagate.
    5. Generate PROJECT-STATUS.md via `generate_project_status()`.
    6. Distribute via `distribute_project_status()`.

    `async def _check_agent(self, agent_id: str) -> None`:
    - Wrap in try/except per D-01. Log exceptions, never propagate.
    - Run `check_liveness()`. If dead, call `on_agent_dead` callback.
    - Run `check_stuck()`. If stuck, call `on_agent_stuck` callback.
    - Run `check_plan_gate()` with stored mtimes. Update stored mtimes. If new plans, call `on_plan_detected` for each.
    - Use `asyncio.to_thread()` for blocking git operations within the async context (per Pitfall 2).

    `def stop(self) -> None`: Sets `_running = False`.

    Tests in `tests/test_monitor_loop.py`:
    - Use pytest-asyncio (`@pytest.mark.asyncio`).
    - Mock checks module functions and TmuxManager.
    - Mock agents.json loading.
    - Verify callbacks fire at correct times.
    - Run single cycle (not full loop) for most tests.
