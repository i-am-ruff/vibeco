  <action>
    Create `src/vcompany/orchestrator/preflight.py`:

    1. Data models (using Pydantic for JSON serialization):
       - `PreflightResult(BaseModel)`: test_name (str), passed (bool), inconclusive (bool = False), details (str), duration_seconds (float).
       - `MonitorStrategy(str, Enum)`: STREAM_JSON = "stream_json", GIT_COMMIT_FALLBACK = "git_commit_fallback".
       - `PreflightSuite(BaseModel)`: results (list[PreflightResult]), strategy (MonitorStrategy), run_at (datetime), claude_version (str = "unknown").
         - `@property all_passed -> bool`: all(r.passed for r in results).
         - `def summary() -> str`: formatted multiline output with test name, pass/fail/inconclusive, details.

    2. `determine_monitor_strategy(results: list[PreflightResult]) -> MonitorStrategy`:
       - Find stream-json test result. If passed=True -> STREAM_JSON. Else -> GIT_COMMIT_FALLBACK. Per D-17, PRE-03.

    3. Four test functions (per D-16, PRE-02). Each returns PreflightResult. Each uses tempfile.TemporaryDirectory() for isolation (per D-15). Use generous timeouts (per research recommendation).

       a. `test_stream_json_heartbeat(timeout: int = 60) -> PreflightResult`:
          - Run: `claude -p --output-format stream-json --dangerously-skip-permissions --max-turns 1 "echo hello"` in tmpdir.
          - Use subprocess.Popen with stdout=PIPE to read output as it arrives.
          - Count JSON lines received. If >= 1 valid JSON event within timeout -> passed=True.
          - On timeout without any events -> passed=False.
          - On subprocess error -> inconclusive=True.

       b. `test_permission_hang(timeout: int = 30) -> PreflightResult`:
          - Run: `claude -p --max-turns 1 "list files"` in tmpdir (WITHOUT --dangerously-skip-permissions).
          - If it completes within timeout -> details="exits normally without permission flag".
          - If it times out -> details="hangs without permission flag -- flag is required".
          - Either way, passed=True (we're documenting behavior, not testing correctness). The details inform ops.

       c. `test_max_turns_exit(timeout: int = 60) -> PreflightResult`:
          - Run: `claude -p --dangerously-skip-permissions --max-turns 1 "list all files in current directory"` in tmpdir.
          - Expect non-zero exit code (per Pitfall 3, research finding). Per PRE-02.
          - passed=True if process exits (regardless of code) and exit code is captured.
          - Record exit code in details for crash classifier to use.

       d. `test_resume_recovery(timeout: int = 120) -> PreflightResult`:
          - Step 1: Run claude with --dangerously-skip-permissions --max-turns 1 -p "create a file named test.txt" in tmpdir.
          - Step 2: Run claude with --dangerously-skip-permissions --continue -p "/gsd:resume-work" in same tmpdir.
          - passed=True if step 2 completes without error. Details include whether session continued.
          - If either step fails -> check if error is timeout (inconclusive) or real failure (passed=False).

    4. `run_preflight(output_path: Path | None = None) -> PreflightSuite`:
       - Runs all 4 tests sequentially.
       - Determines monitor strategy from results.
       - If output_path provided, writes results JSON via write_atomic. Per D-17.
       - Returns PreflightSuite.

    Write tests FIRST for result interpretation and monitor strategy determination (unit tests, no live Claude). The 4 live test functions themselves cannot be unit-tested (they require real Claude Code + API key). Mark any live tests with `@pytest.mark.slow`.
