Output: AgentManager orchestrator class, three CLI commands (dispatch, kill, relaunch), integration tests.

  <action>
    Create `src/vcompany/orchestrator/agent_manager.py`:

    1. `DispatchResult` dataclass: success (bool), agent_id (str), pane_id (str = ""), pid (int | None = None), error (str = ""). Follow GitResult pattern from Phase 1.

    2. `AgentManager.__init__(self, project_dir: Path, config: ProjectConfig, tmux: TmuxManager | None = None)`:
       - Stores project_dir, config. Creates TmuxManager if not injected (for testing).
       - agents_json_path = project_dir / "state" / "agents.json"
       - Loads existing AgentsRegistry from agents_json_path if exists, else creates empty.

    3. `dispatch(self, agent_id: str, resume: bool = False) -> DispatchResult`: Per D-01, D-03, D-04, D-05.
       - Find agent in config.agents by id. Return error if not found.
       - Session name: f"vco-{config.project}" (discretion: naming convention).
       - Get or create tmux session. Create pane with window_name=agent_id.
       - Build env+command as single chained string (per Pitfall 2): `export DISCORD_AGENT_WEBHOOK_URL='{webhook}' && export PROJECT_NAME='{project}' && export AGENT_ID='{agent_id}' && export AGENT_ROLE='{role}' && claude --dangerously-skip-permissions --append-system-prompt-file {prompt_path} -p '/gsd:new-project'`
       - If resume=True: use `-p '/gsd:resume-work'` instead of `/gsd:new-project`. Per D-03, D-07.
       - Send via tmux.send_command(pane, chained_cmd).
       - Extract pane PID (pane.pane_pid). Note: this is the shell PID; Claude PID will be its child.
       - Update AgentsRegistry with new AgentEntry(status="running", launched_at=now). Write atomically. Per D-05.
       - Return DispatchResult(success=True, ...).

    4. `dispatch_all(self) -> list[DispatchResult]`: Per D-02.
       - Create tmux session: f"vco-{config.project}".
       - For each agent in config.agents: create_pane with window_name=agent.id, dispatch.
       - Create one additional pane: window_name="monitor" (for Phase 3 monitor process).
       - Return list of DispatchResult.

    5. `kill(self, agent_id: str, force: bool = False) -> bool`: Per D-06, D-08.
       - Look up agent in AgentsRegistry. Return False if not found.
       - Find child PIDs of the shell PID using /proc/{pid}/stat (per Pattern 4 from research).
       - Verify PID belongs to claude/node process by reading /proc/{pid}/cmdline (per Pitfall 1).
       - If not force: os.kill(pid, SIGTERM), poll for exit up to 10s. If still alive: os.kill(pid, SIGKILL).
       - If force: os.kill(pid, SIGKILL) immediately.
       - Fall back to tmux.kill_pane() if signal delivery fails.
       - Update AgentsRegistry: status="stopped". Write atomically. Per D-08.

    6. `relaunch(self, agent_id: str) -> DispatchResult`: Per D-07, D-08.
       - Call self.kill(agent_id).
       - Call self.dispatch(agent_id, resume=True).
       - Return dispatch result.

    7. Private helpers:
       - `_load_registry(self) -> AgentsRegistry`: load from JSON file or return empty.
       - `_save_registry(self)`: serialize and write_atomic.
       - `_find_child_pids(parent_pid: int) -> list[int]`: scan /proc for children.
       - `_verify_pid_is_claude(pid: int) -> bool`: read /proc/{pid}/cmdline, check for "claude" or "node".
       - `_kill_process(pid: int, timeout: int = 10, force: bool = False) -> bool`: SIGTERM/wait/SIGKILL logic.

    For tests: Mock TmuxManager (inject via constructor). Use tmp_path for project_dir. Create minimal agents.yaml config in fixtures. For kill tests, mock os.kill and /proc reads.
